## 摘要

针对当前人工智能大模型依赖云端服务器、网络依赖性强、隐私性不足、部署成本高的行业痛点，本项目依托以安卓系统为底层移动端原生 Termux 终端环境，实现 Google Gemma AI 轻量化模型全离线本地部署。
本方案无需电脑设备、无需云端算力支持，仅依靠普通以安卓系统为底层的智能手机等移动端硬件资源，通过环境适配、轻量化 AI 模型、移动端部署等核心技术，完成  Google Gemma AI 轻量化模型的环境适配、部署、运行与 API 服务搭建。
项目成功解决了移动端原生 Termux 终端环境实现轻量化 AI 模型部署等技术难题，实现了手机端离线智能问答、文本生成、语义理解等 AI 核心功能。相较于传统云端 AI 服务，本方案具备零网络依赖、数据本地存储、低成本、高隐私性、便携性强的优势，为轻量化 AI 模型移动端落地、老旧移动设备算力再利用提供了全新的低成本解决方案，可广泛应用于离线智能辅助、轻量化 AI 助手、个人隐私算力终端。

**关键词**：Termux；移动端部署；Google Gemma 模型；本地 AI 推理；模型量化；离线算力

## 一、项目背景与研究意义
### 1.1 项目背景
随着生成式人工智能技术的快速迭代，Gemma 作为 Google 推出的轻量化开源大语言模型，凭借参数规模小、推理效率高、泛化能力强、开源免费的特点，成为轻量化边缘 AI 部署的优选模型。当前主流 AI 模型应用模式以云端部署、在线调用为主，存在明显短板；依赖稳定网络环境，无网络场景下无法使用；用户交互数据上传云端，存在隐私泄露风险；云端算力调用存在服务费，长期使用成本较高；部署门槛高，依赖高性能服务器、电脑设备，难以普及。与此同时，智能手机硬件性能持续升级，普通安卓设备已具备基础边缘算力，但大量闲置、老旧手机的算力资源长期处于浪费状态。Termux 作为安卓平台原生开源终端模拟器，无需手机 ROOT 权限，可完整模拟 Linux 运行环境，支持编译、运行开源程序，为移动端本地部署 AI 大模型提供了原生环境支撑。基于以上现状，本项目开展基于原生 Termux 的 Gemma AI 模型移动端本地部署研究，突破云端 AI 的场景限制，挖掘移动终端边缘算力价值。

> **说明**：由于大赛要求 Edge AI 主题采用 E2B/E4B 规格模型完全离线部署在手机、树莓派或嵌入式硬件上（需提供真实硬件演示），由于本人测试手机终端使用运行内存为 8GB、存储空间为 256GB 的华为系列手机为测试环境介于测试环境限制,本人降级使用 Google Gemma 2:2b 轻量化模型进行部署测试。我相信使用高配的智能手机来运行 Google Gemma 4 : E2B/E4B 规格模型应该是没有任何问题。

### 1.2 研究意义
#### 1.2.1 理论意义
本项目探索了轻量化 AI 模型在移动端 Linux 模拟环境的适配与优化方案，完善了边缘端低算力设备部署 AI 模型的技术体系，验证了非 ROOT 安卓系统为终端运行通用 AI 模型的可行性，为轻量化 AI 模型的移动端适配资源优化提供了可复用的技术范式。
#### 1.2.2 实践意义
1.**低成本算力落地** ：盘活闲置安卓手机硬件资源，无需服务器、电脑等专业设备，普通人即可实现个人离线 AI 算力搭建，大幅降低 AI 技术使用门槛。
2.**高隐私安全保障** ：所有数据交互、模型推理全程在手机本地完成，无云端数据上传，彻底规避隐私泄露风险，适配私密办公、离线学习等场景。
3.**场景适配性极强**：摆脱网络束缚，可在无网络、弱网络环境下稳定使用 AI 功能，适用于户外作业、野外调研、应急辅助等特殊场景。
4.**技术普及价值** ：全程原生环境部署、操作轻量化、流程标准化，为青少年 AI 科创、个人技术实践、轻量化 AI 项目开发提供实操参考。
## 二、相关技术概述
### 2.1 Termux 技术简介
Termux 是一款基于安卓平台无 ROOT 权限的开源终端模拟器，可在移动端构建完整的 Linux 运行环境，支持 `apt/pkg` 包管理、代码编译、环境配置、后台进程运行等核心功能。区别于传统移动端虚拟机，Termux 原生适配安卓系统架构，资源占用低、兼容性强，可直接调用手机 CPU、内存等硬件资源，是移动端边缘开发、轻量化服务部署的核心工具，完美适配低资源环境下的程序编译与运行。
> **说明**：Termux是一个适用于 Android 的终端模拟器，其环境类似于 Linux 环境。无需Root或设置即可使用。Termux 会自动进行最小安装 - 使用 APT 包管理器即可获得其他软件包。
1.安全性:使用来自 OpenSSH 的 ssh 客户端访问远程服务器。Termux 将标准的软件包与仿真终端结合在一个完美的开源解决方案中。
2.功能丰富 :你可以在 bash、fish 或 zsh 和 nano、Emacs 或 Vim 之间选择。通过 Grep 搜索您的短信收件箱。使用 curl 访问 API 并使用 rsync 将联系人列表的备份存储在您的远程服务器上。
2.可玩性 :通过 Debian 和 Ubuntu GNU/Linux 等从已知的 APT 包管理器中安装您所想要的。
5.准备扩大规模: 如果需要，连接蓝牙键盘并将您的设备连接到外部显示器 - Termux 支持键盘快捷键并具有完整的鼠标支持。
6.可修补:通过使用 Clang 编译 Go、Rust、Swift 或 C 文件进行开发， 并使用 CMake 和 pkg-config 构建您自己的项目。

### 2.2 Gemma 模型技术简介
Google Gemma 是 Google 开源的轻量化大语言模型，基于 Transformer 架构开发，包含 2B、4B 等轻量参数版本，兼顾推理精度与运行效率。相较于 LLaMA 等同类轻量化模型，Gemma 针对低算力设备做了深度优化，支持量化压缩、动态上下文调节、低内存推理，适配边缘终端设备。
**Gemma 4 系列**：谷歌 DeepMind 于2026年4月发布的开源多模态模型系列。基于Gemini 3 同源技术，4 种架构各有 base 和it 版本，共 8 款模型，采用 Apache 2.0 许可证。
| 架构 | 参数规模 | 版本类型 |
|------|----------|----------|
| E2B  | 2B       | base / -it |
| E4B  | 4B       | base / -it |
| 26B-A4B |   26B | base / -it |
| 31B |       31B | base / -it |
> **说明**：由于本人的测试手机运行内存只有 8GB 的限制，本项目选用了 Gemma 2: 2B 轻量化版本模型，在保证基础文本生成、问答、语义分析能力的同时，大幅降低内存占用，适配了本次测试手机终端的有限硬件资源。

**Gemma 2 系列**：Gemma 2B 是谷歌 DeepMind 于 2024年8月1日发布的轻量级开源人工智能模型，隶属于Gemma系列第二代产品，参数规模为20亿（2B）。该模型与旗舰模型Gemini共享技术基础架构，是Gemma系列中的小参数版本，后续补充了该系列的9B和27B版本。该模型面向设备端设计，采用滑动注意力机制和Logit软封顶技术，提供预训练基础版及指令微调版，使用条款允许各类规模机构进行商业应用与二次分发。2024年8月推出的第二代版本通过蒸馏学习技术实现性能提升，在 LMSYS Chatbot Arena 评测中超越所有GPT-3.5模型。新版集成 Shield Gemma 安全分类器与 Gemma Scope 可解释性工具，支持Keras/JAX框架并在 Google Colab 提供免费运行环境。该模型适配边缘设备运行，可通过 Lora 微调技术实现中文指令优化，已在智能家居、自动驾驶领域展开应用探索。
> **说明**：
Gemma 2B 是 Google 推出的开源大型语言模型系列中的轻量级成员，其规格参数旨在实现高效部署。Gemma 2B 拥有20亿（2B）参数 ，在后续的Gemma 2系列中，Gemma 2 2B的参数规模为26亿（2.6B）。模型主要基于来自网络文档、数学和代码的英文数据进行训练，训练数据量达2万亿（2T）token ，并支持8192个token的上下文长度 。提供预训练基础模型（gemma-2b）和经过指令调优的模型（gemma-2b-it）。其架构基于Transformer解码器，进行了多项改进，包括使用多查询注意力机制、旋转位置嵌入（RoPE）、GeGLU激活函数，以及对每个Transformer子层的输入和输出进行归一化 。这些规格特点使Gemma 2B适用于资源有限的环境，如笔记本电脑、台式机及移动设备。

### 2.3 Ollama.app 核心技术简介
Ollama 是一个开源工具，可在本地快速部署和运行大型语言模型，同时保障数据隐私。
**Ollama 的核心功能**: Ollama 允许用户在本地计算机上运行各种大型语言模型（LLM），无需依赖云端服务，从而保护数据隐私并降低使用成本。Ollama 使得在本地部署大语言模型变得简单高效，用户可以像使用本地软件一样管理和运行 AI 模型，实现 私有化、自由化的 AI 生产力。 其主要特点包括：
1.一键部署主流开源模型：支持 DeepSeek、Qwen、Llama 3、Gemma 等。 完全本地运行：数据不上传云端，适合处理敏感信息。 兼容 OpenAI API 生态：可与现有 API 集成，方便开发者使用。 提供 REST API 和 CLI 工具：支持创建、运行、管理模型，以及多行输入和自定义提示。
2.安装与配置:Ollama 支持 Windows、Linux、macOS 平台。安装完成后，可通过命令 ollama --version 验证是否成功。常用命令包括：
ollama create：从 Modelfile 创建模型
ollama run：运行模型
ollama pull / push：从注册表拉取或推送模型
ollama list / ps：查看本地模型或运行状态
ollama serve：启动服务，无需桌面应用
3.环境变量建议:
OLLAMA_HOST：设置为 0.0.0.0 可允许外部访问，但需注意安全。
OLLAMA_MODELS：指定模型下载路径，避免占用系统盘空间。
4.模型管理与自定义:
Ollama 支持从 GGUF 文件导入模型，并可通过 Modelfile 自定义提示。例如，可以将 Llama3.2 模型设置为“你是马里奥，只以马里奥身份回答”。此外，Ollama 提供预构建模型库，可直接下载和使用。
5.硬件要求:
7B 模型：至少 8 GB RAM
13B 模型：16 GB RAM
33B 模型：32 GB RAM
6.应用场景:
本地 AI 助手：支持 PDF RAG、语音聊天、图像交互等多模态功能。
开发生成式 AI 应用：可与 Python、GoLang 等语言集成，构建聊天机器人或智能工具。
企业数据安全：适合处理敏感数据，避免云端泄露风险。
### 2.4 核心配套技术
1.**Gemma2:2b 轻量化模型技术**:arch gemma2 · parameters 2.61B · quantization Q4_0
2.**本地模型部署技术:** 通过 Termux 安装专属 ollama.app 进行模型部署。

## 三、项目总体设计与技术架构
### 3.1 设计目标
1.基于原生Termux 环境，完成 Gemma 模型全流程本地化部署，实现无网络离线运行。
2.实现文本问答、创意写作、语义解析、逻辑推理等基础 AI 功能正常输出。
3.搭建本地局域网服务，支持本机及局域网设备调用，实现轻量部署应用。
4.形成标准化、可复刻的移动端大模型部署流程，具备实操性和推广性。
### 3.2 整体技术架构
本项目采用**环境层 - 框架层 - 模型层 - 服务层 - 应用层**五层架构设计，全程基于移动端原生环境搭建，架构轻量化、层级清晰：
|  层级  | 名称 | 功能描述 |
|--------|------|----------|
| 环境层 | 安卓原生系统 + Termux 终端 | 构建基础 Linux 运行环境，配置系统权限、编译工具、依赖库 |
| 框架层 | Ollama.cpp 轻量化推理框架 | 适配 ARM 移动端架构，完成源码编译与环境适配 |
| 模型层 | Gemma2:2b 轻量化模型|通过ollama.app进行模型拉取，适配移动端低算力推理场景 |
| 服务层 | 本地局域网服务  | 配置本地端口，启动局域网服务，支持局域网访问 |
| 应用层 | CLI 终端交互 + 局域网共享 | 通过终端交互式问答、多设备共享 AI 算力
### 3.3 技术创新架构优势
区别于传统电脑端部署方案，本架构完全适配移动端硬件特性，通过突破安卓系统Linux权限限制、硬件算力瓶颈，实现轻量化、低成本、离线化部署。

## 四、详细部署实施流程
本项目全程在原生安卓 Termux 环境操作，无需 ROOT、无需电脑、无需外网持续连接。
### 4.1 前期环境准备
#### 1. Termux 安装与初始化
通过 F-Droid 渠道安装正版 Termux （摒弃应用商店停更旧版本，规避兼容问题），打开终端执行:
授予移动设备存储访问权限:
```bash
$ termux-setup-storage
```
更换为国内镜像源(加速下载)
```bash
$ termux-change-repo
```
基础环境更新命令(更新软件包列表,升级所有已安装的软件包)，同步系统依赖源：
```bash
$ pkg update && pkg upgrade -y
```
该步骤完成系统包更新，修复老旧依赖漏洞，为后续编译工作铺垫基础环境。
#### 2. 安装核心编译依赖
移动端编译需要 `git`、`python`、`nodejs`、`wget` 、`curl`、`clang`、`make`、`cmake`、`openssh` 等工具，批量安装编译必备依赖库：
```bash
# 安装核心编译依赖
$ pkg install -y git python nodejs wget curl clang make cmake openssh
```
检查安装依赖版本号,确定安装成功
```bash
# 验证安装软件版本号
$ python --version
```
```bash
# 验证安装软件版本号
$ node--version
```
```bash
# 验证安装软件版本号
$ npm--version
```
```bash
# 验证安装软件版本号
$ git--version
```
### 4.2 推理框架源码编译
采用适配移动端的 `Ollama.cpp` 推理框架（原生支持Gemma轻量化模型、ARM 架构、低内存推理），拉取最新源码并编译，适配 Gemma 模型特殊 token 逻辑与架构特性：
安装适配移动端(Ternux专用版) `Ollama.cpp` :
```bash
# Ternux专用版 OLLAMA
$ pkg install -y ollama
```
启动 `Ollama.cpp` 推理框架服务:
```bash
# Ternux专用版 OLLAMA
$ ollama serve
```
新建 session 从注册表拉取或推送模型:

```bash
# 拉取最新模型
$ ollama pull Gamma2:2b
```
### 4.3 模型本地启动与运行
进入模型目录，执行启动命令，加载量化模型并开启终端交互模式：

```bash
# 启动运行模型
$ ollama run gamma2:2b
```

启动成功后终端提示模型加载完成，可直接输入指令进行 AI 交互。
### 4.4 本地局域网服务部署
为实现局域网服务化调用，搭建本地局域网服务：

```bash
# 启动OPENSSH服务
$ sshd
```
```bash
# 启动OPENSSH服务默认端口
$ sshd -p 8022
```
```bash
# 启动OPENSSH服务进行Termux登录
$ ssh -p 8022 <username>@<mobile_IP>
```

## 五、问题排查与优化方案
| 问题类型 | 问题原因 | 优化方案 |
|----------|----------|----------|
| 编译失败 | 依赖缺失、源码版本过低、移动端架构不兼容 | 更新 Termux 至最新版本，重装全套编译依赖，使用 ollama.app 最新软件，开启 ARM 专属编译参数 |
| 模型加载闪退、内存溢出 | 手机内存不足、量化版本过高、上下文设置过大 | 选用 4bit 轻量量化模型，缩减上下文长度，关闭手机后台应用 |
| 局域网无法访问OPENSSJ服务 | 端口绑定受限、手机防火墙拦截 | 关闭手机本地防火墙，确认设备处于同一局域网网段 |

## 六、功能测试与结果分析
### 6.1 测试环境
- **硬件设备**：普通安卓智能手机（8GB+128GB）

- **测试场景**：离线无网络环境、局域网多设备调用
### 6.2 测试项目与结果
1. **环境稳定性测试**：连续运行模型 8 小时，无闪退、无进程中断，后台运行稳定，适配移动端硬件极限。
2. **基础功能测试**：完成智能问答、文本创作、代码辅助、语义分析、逻辑推理等功能测试，输出内容逻辑通顺、准确率高，符合轻量化大模型使用标准。
3. **离线能力测试**：全程关闭移动数据、WiFi 网络，模型可正常加载、推理、交互，实现 100% 离线运行。
4. **服务调用测试**：局域网内电脑、平板可通过SSH服务调用移动端设备本地服务，满足轻量化应用需求。
### 6.3 测试结论
本项目搭建的基于 Termux 的移动端 Gemma 本地部署方案运行稳定、功能完整、适配性强，完全实现离线 AI 推理、本地算力服务化应用，各项指标达到轻量化边缘 AI 部署标准，方案可行、可复刻、可落地。

## 七、项目创新点
1. **原生轻量化部署创新**：无需 ROOT、无需电脑、无需云端算力，依托手机原生 Termux 环境完成大模型全流程部署，突破传统 AI 部署的设备壁垒，实现人人可搭建的个人离线 AI 终端。
2. **闲置算力盘活创新**：将普通安卓手机、老旧移动设备转化为独立 AI 算力终端，实现闲置硬件资源的智能化再利用，降低 AI 落地硬件成本。
3. **隐私安全架构创新**：全程本地数据处理、无云端上传、零网络依赖，从底层解决 AI 交互数据隐私泄露问题，适配私密化 AI 应用场景。
4. **移动端适配优化创新**：针对安卓系统权限限制、内存短板、架构特性，形成一套专属的编译、量化、加速、排错优化方案，解决了移动端大模型部署的核心痛点。
5. **服务化共享创新**：搭建局域网 API 服务，实现单设备部署、多设备共享算力，大幅拓展移动端本地 AI 模型的应用场景。

## 九、项目不足与未来展望
### 9.1 现存不足
1.受移动端硬件算力限制，相较于云端模型，复杂逻辑推理、长文本生成精度略有不足。
2.高参数 4B 版本模型在低配手机上运行仍存在轻微卡顿，适配机型存在一定硬件门槛。
3.目前仅支持文本生成功能，暂未适配 Gemma 多模态视觉、音频推理能力。

### 9.2 未来展望
1.**功能拓展**：优化多模态适配，实现图片解析、音频处理等多功能落地，完善 Gemma 模型全能力应用。
2.**性能优化**：引入更高效的量化算法、内存调度机制，进一步降低硬件门槛，适配更低配置移动设备。
3. **界面优化**：开发简易移动端可视化交互界面，替代纯命令行操作，降低使用门槛。
4. **场景拓展**：结合物联网、移动端小程序，打造轻量化离线 AI 应用工具包，实现商业化、生活化落地。
